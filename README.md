# AI Intern - Task 1

Този проект има за цел да автоматизира процеса по извличане на информация от протоколи на срещи и съхраняването им в облачна база данни.

## Функционалности
- **Автоматично обхождане:** Скриптът претърсва директория `data/` за проекти и техните срещи.
- **Обработка на документи:** Четене на съдържанието на `.docx` файлове с помощта на `python-docx`.
- **Интеграция с облак:** Съхранение на метаданни и транскрипции в **Supabase** (PostgreSQL).
- **Сигурност:** Използване на `.env` за управление на API ключове.

## Технологичен стек
- **Език:** Python 3.12
- **База данни:** Supabase
- **Библиотеки:** `supabase-py`, `python-dotenv`, `python-docx`

## Задача 2: AI Извличане на данни (LLM Pipeline)

### Стратегия за Prompt
Използвах изрична (zero-shot) стратегия с ясно зададена роля ("expert AI meeting assistant"). Подадох стриктна JSON схема директно в prompt-а. За да избегна грешки във форматирането, изрично инструктирах модела да връща само валиден JSON без Markdown блокове и му зададох правила за крайни случаи (напр. да връща празен масив `[]`, ако няма задачи).

### Валидация и поправка на резултатите
Пайплайнът е защитен срещу халюцинации на модела и мрежови грешки:
- **Предварителна валидация:** Транскрипции под 10 думи се пропускат автоматично.
- **Поправка на формата:** Ако моделът върне Markdown тагове, скриптът ги изчиства преди парсване.
- **Поправка на схемата:** Ако AI пропусне задължителен ключ (напр. `action_items`), скриптът автоматично добавя празен списък.
- **Премахване на дубликати:** Задачите (`action_items`) се филтрират логически, за да се премахнат повтарящи се такива.
- **Мрежова защита:** Добавено е експоненциално изчакване (до 3 опита) при временни грешки в API-то на Google.
